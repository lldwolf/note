第四章 ORACLE的简单数据库操作

4.1 创建数据库

运行“开始”->“Oracle - OraHome92”->“Configuration and Migration Tools”->“Database Configuration Assistant”或直接在命令行下键入：

dbca

这个速度非常慢，要有耐心。我的机器配置为P4 2.4G，512M内存，该过程需耗时半小时左右。

数据库创建完成后，需要为新数据库的SYS和SYSTEM用户指定口令：

数据库至此创建完成。如果打开Windows服务列表，你会发现多了一个服务“OracleServiceTEST2”。

对于新创建的数据库，在OEM的数据库列表中中并不显示出来，一般重启一下“OracleOraHome92Agent”，然后再刷新服务器节点即可，如果还不行的话使用如下方法：

  1. 先启动智能代理服务（即“OracleOraHome92Agent”服务）
  2. 启数据库服务（本例中为“OracleServiceTEST2”服务）
  3. 再启“OracleOraHome92ManagementServer”服务
  4. 刷新结点

4.2 删除数据库

运行“开始”->“Oracle - OraHome92”->“Configuration and Migration Tools”->“Database Configuration Assistant”或直接在命令行下键入：

dbca

启动后画面如下：

这是最后一步确认了，现在点了“是”之后就没得后悔药吃了。

数据库删除后，在OEM中仍然保留着该数据库节点，需手工删除该节点后，再重新刷新服务器节点。有时刷新前需重启一下“OracleOraHome92Agent”服务。
