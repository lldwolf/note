Installing InfoMaker deployment files

When you install InfoMaker, shared files that are used by both PowerBuilder and InfoMaker are installed in the Shared\PowerBuilder directory. You need to deploy a subset of these files with your application.

The following table indicates which feature each file is required to support. For example, PBVM80 .DLL is required for all 32-bit deployed applications. You need not install some other files unless your application uses the features indicated. You should install the deployment DLLs in the same directory as the application or in a directory on the system path.

Database connections The deployment files needed to enable database connections are described in "Making the data source available".

Required for Name
All PBVM80 .DLL, LIBJCC.DLL
Reports and forms PBDWE80 .DLL
Rich Text PBRTC80 .DLL
DataWindow plugin NPDWE80 .DLL
Font mapping PBFNT80.INI
Label report presentation style predefined formats PBLAB80.INI
Database connection tracing PBTRA80 .DLL

Localized deployment files

Localized deployment files are provided for French, German, Italian, Spanish, Dutch, Danish, Norwegian, and Swedish. These files are usually available shortly after the general release of a new version of InfoMaker.

The localized deployment files let you deploy InfoMaker applications with standard runtime dialog boxes in the local language. They handle language-specific data when the application runs, including:

  • DayName function manipulation The DayName function returns a name in the language of the deployment files available on the machine where the application is run.
  • DateTime manipulation When you use the String function to format a date and the month is displayed as text (for example, the display format includes "mmm"), the month is in the language of the deployment files available when the application is run.
  • Error messages InfoMaker error messages are translated into the language of the deployment files.
