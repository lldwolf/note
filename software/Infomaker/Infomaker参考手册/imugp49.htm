定义查询

一个查询就是一条SQL SELECT语句，在查询Painter中生成，保存时设定其名称，然后就可以重复使用了。

一堆废话，有耐心的就帮我翻一下。Queries save time, because you specify all the data requirements just once. For example, you can specify the columns, which rows to retrieve, and the sorting order in a query. Whenever you want to create a report using that data, simply specify the query as the data source.

定义查询的步骤：

  1. 菜单上的 File>New。

  2. 在对话框中选Database面板。

  3. 选择Query图标并点OK。

  4. 在选择表的对话框选择你需要使用的表（注：最画蛇添足的功能，真不如出一个SQL文本编辑器），并点Open。

    你可以选择列、定义排序方式、分组规则、计算列等等。


有关定义SELECT语句的内容，请参考"使用 SQL Select语句".

预览查询结果

创建查询时，你可以进行预览以确定返回结果是否正确。

预览查询的步骤

  1. 在菜单中选择 Design>Preview。

    InfoMaker以表格形式返回查询结果集。

  2. 你可以在数据库Painter中处理返回的数据。

    包括对数据进行过滤或排序，但不能插入或删除数据。

    有关对数据的处理，请参考第三章, "管理数据库".

  3. 看完后，按Preview按钮返回查询Painter。


保存查询

步骤如下：

  1. 选择菜单项 File>Save Query。

    If you have previously saved the query, InfoMaker saves the new version and returns you to the Query painter.

    If you have not previously saved the query, InfoMaker displays the Save Query dialog box.

  2. 给查询起个名字(参考"查询命名").

  3. （可选）输入对查询的注释。

    These comments display in the Library painter. It is a good idea to use comments to remind yourself and others of the purpose of the query.

  4. 按OK.


查询命名

你爱咋起就咋起。The query name can be any valid InfoMaker identifier up to 40 characters. When you name queries, use a unique name to identify each one. One approach is a 2-part name: a standard prefix that identifies the object as a query (such as q_) and a unique suffix. For example, you might name a query that displays employee data q_emp_data.

有关标识符的信息，请参考 第19章, "标识符".

修改查询

步骤如下：

  1. 选择菜单项 File>Open。

  2. 选择要修改的查询。

  3. 改！
