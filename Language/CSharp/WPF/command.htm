Command

什么是 Command

使用 Command 是出于如下目的：

  1. 分离 Command 的执行（Execute）和调用（Invoke）逻辑
  2. 多个 Command Source 可以绑定到同一个 Command
  3. Command 内部定义了 Command 是否可被执行的支持，如某操作如果不可用，则灰掉相应的按钮或菜单

WPF Command 的4个基本概念

  1. Command: 表示执行动作
  2. Source: 调用（Invoke）Command 的对象
  3. Target: 执行 Command 的对象
  4. Binding: Command 和 Command 实现的绑定
